# AGV 中控系统美国客户现场英语背景

更新时间：2026-05-12

## 已知背景

1. 出差目标：2026 年 8 月左右去美国客户现场部署和交流。
2. 系统：AGV 中控系统，是一个大系统。
3. 涉及内容：AGV、VDA5050 协议，以及其他配套模块。
4. 客户行业：物流、仓储、搬运行业。
5. 工作范围：部署、培训、排障、方案交流可能都有；用户是跟着别人去，具体分工尚未完全确定。
6. 现场环境：客户本地、本地内网、Linux 环境。
7. 最大担忧：听不懂客户说什么；不知道怎么表达；不会发音。

## 学习策略调整

用户英语基础薄弱，因此训练目标不是泛泛提高英语，而是围绕 AGV 现场部署做“任务型英语”。

优先训练顺序：

1. 听不懂时的澄清句。
2. AGV / warehouse / deployment 高频词。
3. Linux / network / service / log 排障句。
4. VDA5050 / order / state / connection / map / route 等业务词。
5. 现场部署 checklist 说明。
6. 会后 written confirmation。

## 现场沟通原则

1. 听不懂不要点头假装懂。
2. 关键问题要求客户打字、发邮件、共享截图或日志。
3. 所有关键结论通过邮件或会议纪要确认。
4. 多用图、表格、日志、命令输出、checklist，减少纯口语压力。
5. 用户跟随团队前往，应提前和同行同事约定：复杂商务或方案问题由英语更强的人主讲，用户负责技术执行、日志确认、步骤说明和问题记录。

## 需要继续补充

1. AGV 中控系统英文正式名称。
2. 主要模块：调度、地图、任务、交通管制、设备管理、监控、报表、接口等是否包含。
3. 使用的部署方式：rpm/deb、Docker、Docker Compose、Kubernetes、手工脚本、ansible 等。
4. 依赖组件：数据库、消息队列、Redis、Nginx、MQTT、VDA5050 broker 等。
5. VDA5050 对接对象：AGV 厂商、客户 WMS/WCS、MES/ERP 等。
6. 现场验收指标：AGV 上线数量、任务成功率、路径规划、充电、避障、异常恢复等。
