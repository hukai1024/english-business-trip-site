# 需要补充给发达的信息

为了把通用计划变成你的专属美国客户现场部署材料，请逐步补充以下信息。不会一次性填完也没关系。

## 一、项目信息

1. 你要部署的系统/产品叫什么？
2. 这个系统主要解决什么问题？
3. 客户是什么行业？
4. 客户现场会有哪些角色参加？例如 IT、业务、管理层、供应商。
5. 你在现场的职责是什么？部署、培训、排障、方案交流，还是全部都有？

## 二、技术环境

1. 部署到云上还是客户本地服务器？
2. 操作系统是什么？Linux / Windows / Kubernetes / 其他？
3. 涉及哪些组件？例如数据库、消息队列、Web 服务、API、AI 模型、客户端。
4. 需要客户提前准备什么？服务器、账号、VPN、网络白名单、数据库、证书、域名？
5. 是否有回滚方案？

## 三、现场流程

1. 预计现场几天？
2. 第一天做什么？第二天做什么？
3. 是否需要做培训或演示？
4. 是否需要客户验收？验收标准是什么？
5. 最担心现场出什么问题？

## 四、英语材料

请把以下内容发给发达，即使是中文草稿也可以：

1. 中文项目介绍。
2. 中文部署步骤。
3. 常见报错或历史问题。
4. 你预计客户会问的问题。
5. 你想说但不会说的中文句子。

## 五、发达可以基于这些信息生成

1. 你的英文自我介绍。
2. 项目英文介绍。
3. 部署流程英文话术。
4. 客户 Q&A。
5. 现场 checklist。
6. 会后邮件模板。
7. 专属技术词汇表。
8. 10 分钟模拟会议脚本。
